Skillsとは何か

Skillsは「Claudeに専門的な作業手順を教える命令書フォルダ」です。
毎回プロンプトに長い指示を書かなくても、SkillをONにするだけで、Claudeがその分野の専門家として動いてくれます。

一言で言うと

SKILL.md というファイルに「何をどうやるか」を書くだけで、
Claudeがそのやり方を習得した状態で動いてくれる仕組み。

なぜ必要なのか

Skillなし

毎回「社内のフォーマットに合わせて、〇〇の観点で、△△の形式で…」と長い指示を書く必要がある。再現性が低く、人によって結果がバラバラ。

Skillあり

「進捗報告を書いて」と言うだけで、社内フォーマット・トーン・必須項目を守った文書が自動生成される。一度作れば誰でも同じ品質。

Skillの構造（ファイルの中身）

Skill フォルダ構成
my-skill/ ├── SKILL.md # 必須: 命令書（YAMLヘッダ + Markdown） ├── scripts/ # 任意: 実行スクリプト（Python等） ├── references/ # 任意: 参照ドキュメント └── assets/ # 任意: テンプレート・画像等

Skillの3層ロード構造

Skillsは効率的に動作するため、3段階で情報を読み込みます：

Progressive Disclosure（段階的読み込み）
Layer 1: メタデータ（name + description） 常に読み込み済み
約100トークン / いつでもコンテキストにある
Layer 2: SKILL.md 本文 Skillが発動したとき
500行以内推奨 / 具体的な手順・ルールが書かれる
Layer 3: 添付リソース（scripts, references等） 必要になったとき
サイズ無制限 / スクリプトは実行するだけで読み込み不要

Skillsはユーザーの発言を見て自動的に判断して発動します：

作り方・使い方

最小構成のSkillを作る（5分でできる）

必要なのはフォルダ1つ + ファイル1つだけです：

SKILL.md --- name: weekly-report description: 社内の週次報告を書くためのSkill。進捗・課題・来週の予定を 社内標準フォーマット（3P形式）で作成する。「週報」「進捗報告」 「週次まとめ」などと言われたら必ずこのSkillを使うこと。 --- # 週次報告Skill ## 出力フォーマット 必ず以下の構成で出力すること： 【Progress（進捗）】 - 今週完了したこと 【Plans（来週の計画）】 - 来週取り組むこと 【Problems（課題）】 - 困っていること・ブロッカー ## ルール - 箇条書きは1行30文字以内 - 専門用語は括弧で英語を併記 - トーンは丁寧だが簡潔に

Skillとして使う3つの方法

方法A: フォルダを直接置く（最速）

プロジェクトの .claude/skills/ ディレクトリにSkillフォルダを入れるだけ。そのプロジェクト内でClaude Codeを使うと自動的に読み込まれる。

方法B: Pluginとしてインストール（公式）

GitHubリポジトリをMarketplaceとして登録し、/plugin install コマンドでインストール。複数プロジェクトに横断適用できる。

方法C: claude.aiのプロジェクト設定（Web版）

claude.aiのプロジェクト設定画面からSkillをアップロード。コードが不要でGUIで完結する。

Claude Codeでの具体的な導入手順

# 1. GitHubリポジトリをMarketplaceに登録 /plugin marketplace add your-org/your-skills-repo # 2. インストール（GUIで選んでもOK） /plugin install my-skill@your-marketplace # 3. 使い方を確認 /skills # 4. あとは普通に話しかけるだけ（自動発動） 「今週の進捗報告を書いて」 → weekly-report Skillが自動で発動

Skillを改善するコツ

description を具体的に書く

Skillが発動するかどうかはdescriptionの質で決まります。
「どんな状況で使うか」まで書くと精度が上がります。

OK例：「週報・進捗報告・週次まとめと言われたら必ずこのSkillを使う」

skill-creator Skillを活用する

公式の skill-creator Skillを使うと、テストケース作成→実行→評価→改善のループをClaudeと一緒に回せます。

RAGとSkillsの違い

よく混同されますが、RAGは「何を知るか」、Skillsは「どうやるか」の違いです。

概念の違い
RAG
知識の検索・注入
vs
Skills
手順・行動の定義
観点 RAG Skills
目的 最新・大量の情報を検索して回答に使う 特定のタスクを決まった手順で実行させる
主な入力 ドキュメント・DB・ベクトルストア SKILL.md（手順書）+ スクリプト
Claudeの役割 検索結果を読んで回答を生成する 書かれた手順に従って行動する
得意なこと 「〇〇について教えて」という質問応答 「〇〇を作って」という作業・生成タスク
更新 データベースを更新することで知識が変わる SKILL.mdを書き直すことで挙動が変わる
スクリプト実行 基本的になし Pythonスクリプト等を実行できる
組み合わせ 併用可能。Skillの中でRAGを呼び出す設計もできる
具体例で理解する：
「社内規程を調べて回答する」→ RAG（規程ドキュメントをベクトル検索）
「社内フォーマットで稟議書を書く」→ Skills（書き方のルールをSKILL.mdに定義）
「最新の競合情報を調べて、社内テンプレートで報告書にまとめる」→ RAG + Skills の組み合わせ

対応プラットフォーム

SkillsはClaude専用の機能です。ChatGPTやGeminiでは使えません。
ただしClaudeの中では3つのプラットフォームすべてで利用可能です。

💻
Claude Code（CLI）
エンジニア向けターミナルツール。最も高度なSkill活用が可能。
  • Pluginとしてインストール
  • プロジェクトごとに自動適用
  • スクリプト実行・ファイル操作も可
  • /pluginコマンドで管理
🌐
Claude.ai（Web版）
ブラウザで使える一般向けUI。GUIでSkillを設定できる。
  • プロジェクト設定からSkillをアップロード
  • 有料プランで利用可能
  • 公式Skillが多数プリセット済み
  • コードを書かずに設定できる
⚡
Claude API
開発者向けAPIアクセス。アプリにSkillを組み込める。
  • Skills API でカスタムSkillを登録
  • APIリクエストにSkillを指定
  • システム開発・自社アプリ組み込みに最適
  • Anthropic公式SkillもAPI経由で利用可
他のAIサービスについて：
Skills（Agent Skills標準）は現在Claude専用です。ただし「Agent Skills」という仕様は agentskills.io でオープンな標準として公開されており、将来的に他のAIが対応する可能性もあります。

Skillsを共有する方法

Skillsはチーム・社内・世界中と共有できます。共有の粒度によって3つの方法があります。

👥
方法1: GitHubリポジトリで共有
SkillフォルダをGitHubに置き、Marketplaceとして登録するだけ。チームメンバーは/plugin installでインストールできる。

社内共有に最適。プライベートリポジトリにすれば社内限定にできる。
🏢
方法2: プロジェクトフォルダごと共有
.claude/skills/ フォルダをプロジェクトのGitリポジトリに含めてコミット。

リポジトリをcloneした人は全員、自動的に同じSkillが使える。チーム開発に最適。
🌍
方法3: 公開Marketplaceとして公開
GitHubをPublicにして公開Marketplaceとして公開。Anthropic公式の anthropics/skills のように、誰でも使える状態にできる。

社外・OSS公開に最適。

社内でSkillsを展開する手順

社内Skillsリポジトリを作成する

GitHub（またはGitLab）にプライベートリポジトリを作り、SKILL.mdフォルダを整理して置く。

Marketplaceとして登録する

Claude Codeで /plugin marketplace add your-org/your-skills-repo を実行。

チームに展開する

社内wikiやSlackに手順を共有。各メンバーが /plugin install を実行するだけで使い始められる。

継続的に改善する

SKILL.mdをGit管理しているので、改善はPRで行いレビューできる。バージョン管理も自動。

社内パイオニアとしてのアドバイス：
まず「自分が毎回やっているルーティン作業」を1つSkillにしてみましょう。 skill-creator Skillを使えば、Claudeが一緒にSkillを設計・テスト・改善してくれます。 小さな成功体験を積んでから社内展開すると説得力が増します。

まとめ

Skillsとは

SKILL.mdというファイルに書いた手順書。Claudeが「専門家モード」で動くようになる。3層構造で効率的に動作。

作り方

フォルダ1つ+SKILL.md1ファイルで完成。nameとdescriptionが鍵。skill-creatorで一緒に作れる。

RAGとの違い

RAGは「何を知るか」、Skillsは「どうやるか」。目的が異なり、組み合わせて使うことも可能。

プラットフォーム・共有

Claude専用（Code / Web / API）で動く。GitHubで社内・社外に共有可能。Claudeを使う人全員と共有できる。
